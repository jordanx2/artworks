About This Application

This web application allows users to manage a catalog of artworks. It was developed as part of the Enterprise Application Development assignment using a RESTful client-server architecture, and is designed to resemble the experience of a real-world catalog management system.

Technologies Used

  • Frontend: React, TypeScript, SCSS, Vite
  • Backend: Node.js, Express, MongoDB (in-memory via mongodb-memory-server)
  • Data Streaming: stream-json for efficient loading of large JSON files
  • Form Handling & Validation: Formik for form management, Yup for validation
  • API Communication: Axios for structured HTTP requests
  • UI Libraries: React Bootstrap, Font Awesome
  • Notifications: react-toastify for real-time user feedback
  • Exporting: JSZip and FileSaver.js for image export functionality

Key Features

  • Insert, update, delete, and search artworks
  • Client-side form validation using Formik and Yup
  • Paginated navigation through artworks
  • Grid View: View artworks in a tabular format for better readability
  • User Authentication: Login/signup functionality with simple credential checks
  • Collections: Users can add artworks to and remove artworks from their personal collection
  • Export Images: Users can open a modal to select and download one or multiple artwork images
  • Collection Downloads: Users can download all images from their saved collection with a single click
  • Dynamic Image Export: If one image is selected, it downloads directly; if multiple are selected, they are bundled into a ZIP file
  • Smart ZIP Handling: Images are downloaded in proper format using Canvas and organized in a folder
  • Dedicated "About this Page" section served via static HTML
  • Responsive UI with dynamic controls and transitions

Limitations

  • In-memory MongoDB resets on server restart (no permanent data storage)
  • User authentication is basic (no hashing or session management)
  • No real image hosting (only image URLs used)

Possible Enhancements

  • Replace in-memory DB with MongoDB Atlas for persistent storage
  • Add password hashing and session/token management for secure login
  • Allow users to upload and store images
  • Enhance search with filters like classification, date, and artist
  • Add edit/view modes to collections
