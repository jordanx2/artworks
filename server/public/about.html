About This Application

This web application allows users to manage a catalog of artworks. It was developed as part of the Enterprise Application Development assignment using a RESTful client-server architecture.

Technologies Used

  • Frontend: React, TypeScript, SCSS, Vite
  • Backend: Node.js, Express, MongoDB (in-memory via mongodb-memory-server)
  • Data Streaming: stream-json for efficient loading of large JSON files
  • Form Handling & Validation: Formik for form management, Yup for validation
  • API Communication: Axios for structured HTTP requests to the server
  • UI Libraries: React Bootstrap for layout and components, Font Awesome for icons
  • Notifications: react-toastify to provide real-time feedback to the user

Key Features

  • Insert, update, delete and search artworks
  • Client-side validation using Formik and Yup
  • Navigation through artworks

Limitations

  • Currently uses an in-memory MongoDB database which resets on restart
  • Basic search functionality without advanced filters
  • No persistent file uploads or image hosting

Alternative Approaches Considered

  • Using a persistent MongoDB Atlas database for permanent storage
  • Integrating Redux for more complex state management
  • Implementing server-side rendering with Next.js for SEO optimization
